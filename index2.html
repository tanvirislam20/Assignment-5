Question-3: What is event delegation in the DOM?
Event delegation reduces the number of event listeners in the DOM, which can improve performance, especially when working with a large number of elements or dynamically added elements. By handling events at the parent level, memory usage and processing load are reduced.
Question-4: What is event bubbling in the DOM?
Event bubbling directs an event to its target. It works like this: When an element (like a button) is clicked, an event is directed to the element. If an event handler is set for the element, the event handler is triggered. Then the event "bubbles up" to the elements parent. If an event handler is set for the parent, this event handler is triggered. The event continues to bubble, until it reaches the top.
